# Проект «Мои деньги» — приложение учёта финансов (аналог Bluecoins)

Владелица — Алина, не программист. Общайся по-русски, простыми словами, без жаргона.
Она диктует голосом — в сообщениях бывают опечатки и оговорки, интерпретируй по смыслу.

## Что это

- `money.html` — ВСЁ приложение: один самодостаточный файл (HTML+CSS+JS, без сборки и внешних библиотек).
  Работает на GitHub Pages: https://alina3500-2-arch.github.io/avito-plan/money.html
- `ios/` — тонкая iOS-оболочка (Swift, WKWebView), которая просто открывает эту страницу.
  Установлена у Алины через SideStore. Она даёт пуш-уведомления и приём СМС через URL-схему.
- `index.html` — старая страница «Авитолог», НЕ ТРОГАТЬ, к приложению не относится.
- `.github/workflows/ios.yml` — сборка `MoiDengi.ipa` (не подписан, для SideStore),
  публикуется в релиз `ipa-latest`. Собирается ТОЛЬКО при изменении `ios/**` или самого workflow.

## Главное правило обновлений

Правки интерфейса/логики = правка `money.html` → приложение обновится САМО при следующем
открытии (оболочка грузит страницу с Pages). Переустановка .ipa НЕ нужна.
Пересборка .ipa нужна только если менялся код в `ios/` — тогда Алине нужно скачать
https://github.com/Alina3500-2-arch/avito-plan/releases/download/ipa-latest/MoiDengi.ipa
и переустановить через SideStore. Избегай правок `ios/` без необходимости.

## Рабочий процесс (каждая правка)

1. Работай в ветке, которую выдала система (или создай от main).
2. Правь `money.html`. **Обязательно увеличь номер версии** — строка
   `учёт как в Bluecoins · версия N` в шапке бокового меню (drawer). По номеру Алина
   проверяет, что обновление долетело (меню ☰ в приложении).
3. Протестируй в headless-браузере (см. «Как тестировать»), посмотри скриншоты глазами.
4. Закоммить в ветку, запушь, затем смерджи в `main` и запушь main — Pages деплоит
   с main, задержка+кэш до 10 минут.
5. В ответе Алине: что сделано (простыми словами), номер версии, скриншоты (SendUserFile),
   и В КОНЦЕ КАЖДОГО сообщения ссылку: https://alina3500-2-arch.github.io/avito-plan/money.html

## Как тестировать

Playwright + chromium уже стоят (`/opt/pw-browsers/chromium`, модуль в
`/opt/node22/lib/node_modules/playwright`). Открывай `file:///…/money.html`,
viewport 390×844. Примеры готовых сценариев — в истории коммитов не ищи, просто пиши свой:
- СМС-приём: открой `money.html?sms=<urlencoded СМС>` — должна открыться полноэкранная
  форма с суммой/магазином/категорией.
- Тестовая СМС: `МИР-1234 16:32 Покупка 345.50р MAGNIT Баланс: 12 345.67р`
- Диалоги confirm/alert принимай: `page.on('dialog', d => d.accept())`.
- Реальные данные для теста можно залить через «Восстановить из копии» (см. dr-item restore,
  input #impFile) — но файла истории в репо НЕТ (он приватный, у Алины в чате/Файлах).

## Устройство money.html (не ломай!)

- Данные: `localStorage`, ключ `bcMoney.v2`. Структура:
  `{accounts, cats, txs, rems, inbox, merchantMap, seenSms, accV3, savRemV6}`.
  НЕ меняй ключ и не переименовывай поля без миграции со старого формата.
- `inbox` — «Незаписанные операции»: пойманные СМС, которые ещё не записаны
  (телефон был заблокирован, окно смахнули и т.п.). Показываются при открытии
  приложения и из меню ☰. Оболочка дописывает такие СМС в файл
  `Documents/inbox.txt` (автоматизация Команд пишет в «Файлы → На iPhone →
  Мои деньги → inbox.txt») и при запуске передаёт их в `importInboxTexts([тексты])`.
- Счета: `a1` Сбер карта (основной, сюда падают СМС), `a2` Кредитная карта
  (start −13000, limit 80000), `a3` Сберегательный счёт.
- Категории: массив `cats` c `{id,name,icon,type:'expense'|'income',group}`.
  Группы расходов и их порядок ВЕЗДЕ: `GROUPS=['Основное','Нужное','Прочие']`.
- Вкладки (порядок менять нельзя, Алина привыкла): операции → напоминания (будильник) →
  сводка (корзинка) → диаграммы → крупные расходы.
- Глобальные функции, на которые завязана iOS-оболочка — НЕ ПЕРЕИМЕНОВЫВАТЬ:
  - `handleSmsText(text,{fromUrl:true})` — вызывается оболочкой при `moidengi://sms?text=…`
  - `openDueSheet(id)` — вызывается при нажатии на пуш и `moidengi://pay?id=…`
  - `importInboxTexts([тексты])` — вызывается оболочкой при запуске: непрочитанные
    СМС из файла inbox.txt; также шлётся `webkit.messageHandlers.inboxCount`
    (число незаписанных — оболочка ставит/снимает ежедневный пуш в 20:00)
  - `syncNativeReminders()` — шлёт напоминания в `webkit.messageHandlers.reminders`
    (вызывается из `save()`); формат сообщения:
    `[{id,name,amount,due:'YYYY-MM-DD',time:'HH:MM',repeat:'none|weekly|monthly|yearly',dayBefore:bool}]`
  - Определение «мы внутри приложения»: `IN_APP` (userAgent содержит `MoiDengiApp`).
- URL-параметры страницы: `?sms=<текст>` (разбор СМС), `?pay=<remId>` (окно «Пора оплатить»).
- Парсер СМС Сбера: `parseSms()` + `CAT_HINTS` (регэкспы магазин→категория) +
  `merchantMap` (запоминание «магазин→категория» по действиям пользователя).
- Дизайн: копия Bluecoins по скринам Алины — синий #1E88E5, красный расход #E53935,
  зелёный доход #43A047, полноэкранные формы без вертикальной прокрутки, калькулятор
  и календарь всплывающими окошками с затемнением. Русский язык интерфейса.
  Всё должно помещаться в экран iPhone (390×844) без прокрутки в формах.

## Стиль общения с Алиной

- Просто и по-дружески, женский род от первого лица («сделала», «починила»).
- После каждой выгрузки: номер версии + что изменилось + скриншоты + ссылка в конце.
- Напоминание про кэш: обновление видно через 5–10 минут, версию проверять в ☰.
- Не спрашивать разрешения на очевидные шаги — делать и показывать результат.

## Чего НЕ делать

- Не коммитить в репозиторий личные данные (историю операций, bluecoins-perenos.json) —
  репозиторий ПУБЛИЧНЫЙ.
- Не разбивать money.html на несколько файлов, не добавлять сборщики/фреймворки/CDN.
- Не менять bundle id, URL-схему `moidengi://`, имя релиза `ipa-latest`.
- Не трогать index.html.
